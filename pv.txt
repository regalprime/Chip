1. Vòng đời app 
 - resumed : App đang chạy foreground (user đang dùng)
 - inactive : App tạm thời không nhận input
 - paused : App vào background
 - detached : App bị tách khỏi UI
2. Vòng đời Widget

- initState() → chạy 1 lần
- didChangeDependencies()
- build() → có thể gọi nhiều lần
- didUpdateWidget()
- deactivate()
- dispose() → huỷ

4. Bloc Hoạt dộng như nào
- Tách riêng UI với xử lý logic + state
- UI → Event → Bloc → xử lý → State → UI update
5. Widget tree là gì?
- Toàn bộ UI là cây widget lồng nhau.
6. BuildContext là gì?
- Là “vị trí” của widget trong cây widget → dùng để truy cập theme, navigator, provider...
7. Tối ưu hóa Performance
- dùng const widget
- tránh rebuild không cần
- Load và tải ảnh nặng tối ưu hóa performace như nào ?
 + Flutter sử dụng single-thread cho UI (main isolate), nhưng việc decode ảnh được engine xử lý ở background thread nên không block UI. Nếu cần xử lý ảnh nặng bằng Dart thì có thể dùng isolate để tránh lag.
